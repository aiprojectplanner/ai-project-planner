# Feature: Internationalization (i18n)

## Purpose

Allow users to switch the product UI between English (`en`) and Simplified Chinese (`zh`).

## User Flow

1. User clicks **EN** or **中文** on the language control.
2. The selection is stored in `localStorage` under `app.locale`.
3. The document root `lang` attribute updates (`en` or `zh-CN`).
4. All wired screens re-render with the selected locale copy.

## Implementation

- **Provider**: `src/i18n/I18nProvider.jsx` wraps the app in `main.jsx`.
- **Hook**: `src/i18n/useI18n.js` exposes `{ locale, setLocale, t }`.
- **Copy**: `src/i18n/messages.js` holds nested string trees for `en` and `zh`.
- **Switcher**: `src/components/LanguageSwitcher.jsx` (segmented EN / 中文 buttons).

## Placement

- **Authenticated layout**: Language switcher appears in the top `PageHeader` (next to notifications) and in the **sidebar** above the user card.
- **Landing** (`/`): Switcher in the sticky header next to the primary CTA.
- **Auth** (`/auth`): Switcher in the top-right corner of the page.
- **Project Editor**: Switcher in the editor header toolbar (logged-in users also have sidebar controls).

## Scope Notes

- Server/API error messages may still be returned in English from backends; the UI maps common client-side strings where applicable.
- Marketing demo content on the landing page is localized via `messages` (`landing.demos.*`).

## Error Cases

- Missing translation key: `t()` falls back to the English string, then to the key path.

# Deployment Rules 
 
 ## Branch Safety 
 
 Code changes should first go to a development branch. 
 
 Main branch should only contain stable code. 
 
 ## Language Policy (Mandatory) 
 
 - Commit messages must be written in English only. 
 - Pull request titles and descriptions must be written in English only. 
 - Do not introduce Chinese text in commits or PR metadata. 
 
 ## Testing Before Deployment 
 
 Before deploying: 
 
 - build must succeed 
 - no console errors 
 - key features must be tested 
 
 ## Avoid Automatic Production Changes 
 
 AI should never automatically deploy to production without confirmation.
